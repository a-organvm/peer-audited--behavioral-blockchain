# Customer Success

Support playbooks, FAQ content, onboarding sequences, and user satisfaction tracking for Styx.

## What Lives Here

- **FAQ / Help Center articles** — Answers to common user questions, written in our linguistic cloaker vocabulary (no gambling terms)
- **Onboarding sequences** — Email drip campaigns that guide new users through their first week
- **Support playbook** — How to handle common support requests (refunds, account issues, technical problems)
- **Churn signals** — Patterns that indicate a user is about to leave (e.g., "Ostrich Effect" — avoiding the app during difficult periods)
- **NPS tracking** — Net Promoter Score surveys and analysis

## Who Uses This

- **Non-technical co-founder**: You'll handle early support directly. These docs are your reference guide.
- **Technical co-founder**: Building in-app help features, understanding user pain points
- **Support agent (`styx-support`)**: Drafts help articles, analyzes support patterns

## Support Channels

- **Beta phase**: Email + Discord (founder-handled)
- **Launch phase**: In-app help + email + community forum
- **Scale phase**: In-app chatbot + help center + escalation to humans

## Key Metrics

- Response time (target: < 4 hours during beta)
- Resolution rate
- NPS (Net Promoter Score)
- Support ticket volume by category
