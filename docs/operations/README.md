# Operations & Infrastructure

Playbooks, runbooks, and procedures for running Styx in production.

## What Lives Here

- **Incident response runbook** — What to do when something breaks: severity levels, who to call, how to roll back
- **Monitoring setup** — Which services we watch, what alerts mean, how to acknowledge them
- **Deployment procedures** — How code gets from a developer's machine to production
- **Database backup & restore** — How we protect user data and how to recover if something goes wrong
- **Cost management** — Cloud spending guardrails and scaling decisions
- **Load testing results** — Performance benchmarks and capacity planning

## Who Uses This

- **Technical co-founder**: Day-to-day operations reference
- **Non-technical co-founder**: Understanding what "uptime" means, what incidents look like, when to escalate
- **Future hires**: On-call reference, onboarding to production systems

## Key Contacts

- Cloud hosting: Render (render.com)
- Error tracking: Sentry
- CDN/WAF: Cloudflare
- Storage: Cloudflare R2
